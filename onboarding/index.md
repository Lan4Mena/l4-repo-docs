# 🚀 Bienvenida al Equipo de Tecnología — l4 repo docs

¡Hola! Si estás leyendo esto, es porque te estás incorporando al equipo de tecnología de **l4 repo docs**. ¡Te damos una cálida bienvenida! 🎉

Este documento tiene como objetivo guiarte durante tus primeros días para que puedas configurar tus herramientas y accesos de la forma más rápida y autónoma posible.

---

## 📅 Primeros Pasos (Día 1 y 2)

### 1. Canales de Comunicación y Colaboración
Asegúrate de tener acceso e iniciar sesión en las siguientes herramientas clave:
*   **Slack:** Solicita invitación al espacio de la empresa. Únete a los canales `#general`, `#tech-announcements` y `#dev-talk`.
*   **GitHub:** Envía tu usuario de GitHub al responsable de tu equipo para ser agregado a la organización **L4CR** y tener acceso a los repositorios correspondientes.
*   **Gestor de Tareas (Jira / Linear):** Revisa que tengas acceso al tablero de tu equipo de trabajo.

### 2. Entorno Local de Desarrollo
Antes de clonar los repositorios, asegúrate de configurar tu máquina local:
*   **Git:** Configura tu nombre y correo global en Git:
    ```bash
    git config --global user.name "Tu Nombre"
    git config --global user.email "tu.correo@portafolio.com"
    ```
*   **SSH Keys:** Genera y registra tu clave SSH en GitHub para poder clonar y realizar commits de forma segura.
*   **Docker:** Instala Docker Desktop (necesario para levantar bases de datos y servicios en local).

---

## 💻 Configuración de Proyectos

Una vez tengas tus accesos a la organización de GitHub:
1. Dirígete al **[Catálogo de Repositorios](../docs/repositorios.md)**.
2. Identifica el repositorio del proyecto en el cual vas a trabajar.
3. Clona el proyecto y sigue los pasos detallados en su archivo `docs/despliegue.md` o `README.md`.

*Si encuentras algún problema o paso desactualizado durante la configuración de tu entorno, te animamos a corregirlo y enviar un Pull Request para mejorar el proceso para futuros compañeros.*
